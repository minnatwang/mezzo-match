Investor Day Matching Algorithm

CSV UPLOAD INSTRUCTIONS

Your spreadsheet needs to have the following columns:

<

  • Names of the companies and funds/investors (one name per row)
  • This column is what the final schedule will be made from, so if an entity is requested but not in this column, it won't have a schedule

  • stage of the fund or next round the company will be raising; e.g. Seed, A, B
  • Note that this is not actually used in the algorithm and you can leave it blank for all entities

  • If the entity is a company, type is 0; if investor/fund, type is 1

  • A number denoting how important it is that the entity gets the meetings it has requested, where higher number means more important
  • Typically a number between 1-5 or 1-10 (depending on the range you want)

  • These columns hold the main meeting requests
  • The number of choice_# columns should match the total number of possible meetings any one entity can have (e.g. the number of slots in everyone's schedules)

  • These columns hold backup meeting requests
  • You can offer as many or as few backups as you want to attendees


One example of a correct spreadsheet's first row:
entity | type | importance | choice_1 | choice_2 | choice_3 | backup_1
